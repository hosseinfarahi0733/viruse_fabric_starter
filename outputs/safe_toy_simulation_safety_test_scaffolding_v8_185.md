        ## Safe toy simulation safety test scaffolding

### Purpose

This milestone defines safety-test scaffolding before simulator implementation.

The goal is to make unsafe content fail before dynamics code exists.

This is boring infrastructure, which is exactly why it matters. Disasters often begin when someone skips boring infrastructure because the interesting button was shinier.

---

## Safety guard purpose

The safety guard must enforce:

- abstract-only entities,
- synthetic-only fixtures,
- unitless-only variables,
- no real biological datasets,
- no real pathogen simulation,
- no real receptor parameters,
- no operational host targeting,
- no wet-lab protocols,
- no actionable biosafety-risk instructions.

---

## Blocked operational categories

The scaffold must block eight operational category families:

1. Real pathogen identity or parameterization.
2. Real receptor identity or parameterization.
3. Host-targeting design objective.
4. Wet-lab execution wording.
5. Real-world infectivity optimization.
6. Immune evasion optimization.
7. Real host-range prediction.
8. Actionable biosafety-risk instruction.

The report intentionally records categories, not operational examples.

---

## Allowed toy categories

The scaffold allows:

- synthetic graph nodes,
- synthetic graph edges,
- toy agents,
- unitless compatibility scores,
- unitless defense scores,
- unitless transport scores,
- deterministic seeds,
- aggregate observation scores,
- abstract emergence explanations.

These are allowed only when they remain non-operational.

---

## Pre-implementation safety gate

Before implementation, a candidate simulator module must pass:

- prohibited-category scan,
- synthetic-fixture check,
- unitless-variable check,
- abstract-graph check,
- no-real-dataset check,
- no-operational-objective check,
- boundary-counter check.

A simulator must not be implemented until the scaffold exists.

---

## Safety test plan

### Test group A: Synthetic fixture safety

Checks:
- fixture names are synthetic,
- graph nodes are abstract,
- variables are unitless,
- deterministic seeds are present,
- no real biological dataset is imported.

### Test group B: Abstract variable safety

Checks:
- allowed variables are symbolic and unitless,
- variables do not claim real biological meaning,
- no variable is mapped to a real receptor, pathogen, host, tissue, clinical condition, or wet-lab process.

### Test group C: Boundary counter safety

Required zero counters:
- Real pathogen simulation count: 0
- Real receptor parameter count: 0
- Operational host targeting count: 0
- Wet-lab protocol count: 0
- Actionable biosafety-risk instruction count: 0
- Real-world infectivity optimization count: 0
- Immune evasion optimization count: 0
- Real host range prediction count: 0

### Test group D: Architecture continuity

Checks:
- v8.184 architecture is preserved,
- ToyAgent remains symbolic,
- SyntheticCompartment remains abstract,
- SyntheticGraph remains synthetic,
- observation projection remains aggregate and non-predictive.
