# Literature Mapping Plan v1

## Status

Scope: draft-literature-mapping-plan-no-new-citations-no-claim-validation

This artifact creates a literature mapping plan only. It does not add citations, does not add references, does not claim literature support, does not claim validation, and does not claim manuscript readiness.

## Purpose

The purpose of this plan is to identify the bodies of literature that can later be searched and mapped against VF-H2. The plan is designed to support a guarded related-work section without overstating the relationship between VF-H2 and existing work.

VF-H2 should be positioned as a safe abstract toy hypothesis about memory-ledger-driven propagation over finite graph-like state spaces. Literature should be used for conceptual positioning, not as validation evidence.

## Mapping Rule

Future citations may only be added after a separate citation-search and source-verification step. No source is added in this artifact.

Literature mapping must not convert analogy into validation. Similarity to an existing field does not prove VF-H2, does not validate the full Viruse Fabric theory, and does not establish real-world or biological relevance.

## Candidate Literature Buckets

### 1. Graph Dynamical Systems

Purpose:
Map VF-H2 to the broad idea of state updates over graph-structured systems.

Search targets:
- graph dynamical systems
- finite graph state updates
- discrete dynamical systems on graphs
- local update rules on networks

VF-H2 relation:
VF-H2 uses a finite abstract graph and transition rules over graph-like state spaces.

Boundary:
Graph dynamical systems literature can contextualize the formal structure, but it cannot validate VF-H2.

### 2. Memory in Complex Networks

Purpose:
Map the persistent memory ledger to broader work on memory-dependent network dynamics.

Search targets:
- memory effects in complex networks
- history-dependent network dynamics
- non-Markovian network processes
- path-dependent diffusion on networks

VF-H2 relation:
VF-H2 explicitly separates persistent memory-ledger dynamics from no-persistent-memory null controls.

Boundary:
Memory literature may support conceptual framing, not toy-result validation.

### 3. Non-Markovian Propagation Models

Purpose:
Connect VF-H2 to propagation models where future state depends on historical structure.

Search targets:
- non-Markovian spreading processes
- history-dependent spreading
- memory-dependent diffusion
- temporal network memory

VF-H2 relation:
VF-H2 can be framed as a toy non-Markovian propagation mechanism through persistent ledger state.

Boundary:
Do not imply that VF-H2 models real-world contagion, pathogens, or epidemiology.

### 4. Reservoir Computing and Echo State Networks

Purpose:
Map the role of accumulated state and memory-like internal dynamics.

Search targets:
- reservoir computing memory
- echo state networks memory capacity
- recurrent state dynamics
- fading memory property

VF-H2 relation:
VF-H2 and reservoir-style systems both involve historical state affecting future transitions, but VF-H2 is not an echo state network.

Boundary:
This bucket is analogy and positioning only.

### 5. Cellular Automata and Memory-Augmented Automata

Purpose:
Map VF-H2 to discrete rule-based update systems.

Search targets:
- cellular automata with memory
- memory cellular automata
- graph cellular automata
- automata with history-dependent rules

VF-H2 relation:
VF-H2 can be described as a memory-ledger-driven abstract propagation model with discrete update rules.

Boundary:
Do not claim VF-H2 is equivalent to a known cellular automaton class unless formally proved.

### 6. Temporal Networks

Purpose:
Map the three-time coordinate and ledger accumulation to time-indexed network processes.

Search targets:
- temporal networks
- dynamic networks
- time-varying graphs
- multilayer temporal dynamics

VF-H2 relation:
VF-H2 uses local update time, ledger accumulation time, and projection or observation time.

Boundary:
Temporal-network literature can frame the timing structure, not validate the results.

### 7. Causal Sets and Abstract Causal Structure

Purpose:
Map the phrase causal mass and ordered update structure to abstract causal thinking.

Search targets:
- causal set theory abstract order
- causal structure in discrete systems
- partial order dynamics
- causal graph dynamics

VF-H2 relation:
VF-H2 uses a bounded abstract causal-mass functional, but it does not claim to be a causal set theory model.

Boundary:
Avoid overclaiming. Causal set theory should be treated only as distant conceptual context unless a precise formal connection is built.

### 8. Null Models and Ablation in Simulation Studies

Purpose:
Support the methodology of testing against null controls, ablations, and artifact checks.

Search targets:
- null models in network science
- ablation studies in computational modeling
- simulation artifact detection
- robustness checks in computational experiments

VF-H2 relation:
VF-H2 uses no-persistent-memory null controls, ablation tests, artifact-resistance checks, and independent implementation-level replication.

Boundary:
Methodological literature can justify the evaluation style, not validate the theory.

### 9. Artificial Life and Toy Models

Purpose:
Position VF-H2 as an abstract toy-model study rather than biological modeling.

Search targets:
- artificial life toy models
- abstract propagation models
- computational toy models
- emergence in artificial life systems

VF-H2 relation:
VF-H2 is best positioned as a safe abstract toy model in complex systems or artificial life style, not as biology.

Boundary:
Do not move into real biological claims.

## Future Search Protocol

A future source-verification step should:

1. Search each bucket separately.
2. Prefer survey papers, foundational papers, and well-cited methodological papers.
3. Record exact bibliographic metadata.
4. Separate conceptual-positioning citations from method-support citations.
5. Reject sources that would force biological, operational, or real-world claims.
6. Add citations only after verification.
7. Keep a citation boundary table showing what each source can and cannot support.

## Related-Work Section Shape

A future related-work section should use this order:

1. Graph dynamical systems and discrete network dynamics.
2. Memory and non-Markovian processes in networks.
3. Memory-augmented automata and reservoir-style analogies.
4. Temporal and abstract causal structure.
5. Null models, ablation, robustness, and artifact detection.
6. Artificial life and toy-model positioning.

## Explicit Non-Claims

This mapping plan does not claim:

- literature validation,
- theory validation,
- formal proof,
- external validation,
- independent empirical validation,
- biological relevance,
- real-world relevance,
- manuscript readiness,
- submission readiness.

## Safety Boundary

Safe abstract toy model only. No real biological datasets, no real pathogen models, no receptor parameters, no operational targeting, no wet-lab protocol, no infectivity optimization, no immune evasion optimization, and no host range prediction are introduced.
